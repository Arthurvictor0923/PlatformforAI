模型部署 📋 字段说明
📖 模型部署字段说明
模型名称：模型的自定义名称，允许重复，非唯一标识。由创建者在模型广场或数据管理中定义，仅用于展示和搜索。
模型编码：模型的全局唯一标识，创建时指定后不可修改。格式：小写字母开头，仅允许小写字母、数字和连字符（-），如 model-005。用于 API 调用和镜像寻址。
项目组：模型所属的业务团队，用于权限隔离和资源配额管理。
模型类型：模型的算法框架类型，如 AI（通用 AI 模型）、ML（传统机器学习）等。
模型版本：当前部署的模型版本号，格式与模型编码一致：小写字母开头，小写字母+数字+连字符，如 v1-1、v2025-11-28-1。
状态：模型部署生命周期 — 未发布→待部署→部署中→接口校验中→部署成功/部署失败。状态决定可执行的操作。
同步模型广场：标识该模型是否已同步到模型广场供其他用户发现和调用。
调用次数：该模型部署后通过推理接口被调用的累计次数。
演示体验：部署成功后可直接在页面内快速测试模型推理效果。
🚀 部署与推理服务说明
新增推理服务（顶部按钮）：打开模型选择器，选取一个模型后为其创建推理服务（即发布+部署的快捷通道）。适用于从已有模型快速搭建在线推理能力。
行级「发布」按钮：对当前行的模型直接执行发布操作，将其状态从未发布变为待部署，然后可继续部署。
部署触发：点击「部署」按钮后，系统立即将状态置为"部署中"，并异步执行容器部署（Pod 调度 → 镜像拉取 → 容器启动）和接口健康校验。
接口健康校验：部署完成后自动校验 6 个接口。3 个必选接口（/reasoning 单次推理、/health 健康检查、/info 服务信息）必须全部通过。
可选接口：/batch-reasoning（批量推理）、/models（子模型列表）、/warmup（模型预热）失败不影响部署成功。
服务资源监控（顶部按钮）：查看当前已部署服务的集群资源状态，包括 GPU 使用率、内存占用、请求 QPS、P99 延迟等指标。
日志查看：部署中/成功/失败后均可点击「日志」查看部署进度、接口校验结果和完整日志。

🔌 三、接口健康校验说明
接口路径类别功能方法失败影响
/reasoning必选单次推理接口，验证模型核心推理能力POST🔴 部署判定失败

🧪 三、如何体验各状态
操作预期效果
找到"待部署"行 → 点击「部署」→ 确认状态变为"部署中" → 抽屉自动打开 → 时间线推进 → 接口逐个校验 → 80% 概率成功
在日志抽屉中使用"状态预览"下拉框可快速预览抽屉在各状态下的展示效果，无需等待实际部署
找到"部署失败"行 → 点击状态芯片或「日志」查看失败原因、时间线错误节点、接口失败详情
部署失败抽屉中 → 点击「重新部署」二次确认弹窗 → 状态回到"待部署" → 自动弹出部署弹窗
找到"部署成功"行 → 点击「日志」查看完整时间线、接口校验结果、下载日志按钮
