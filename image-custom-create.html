基础信息 📋 字段说明
⚠️
镜像名称
镜像名称是镜像在平台中的唯一标识，创建后不可修改。命名需使用小写字母、数字及连字符 - 组成，长度限制 20 个字符。建议使用能直观反映镜像用途的简短名称，如 pytorch-train、model-infer。名称不可与平台已有镜像重复。
⚠️
镜像分类
镜像分类用于将镜像归入不同的业务领域，方便在镜像市场中按分类检索和筛选。请根据镜像的实际 AI 能力类型选择最匹配的分类：sensing（感知类，如图像识别、语音识别）、decision（决策类，如路径规划、策略优化）、simulation（仿真类，如环境模拟、数据仿真）、llm（大模型类，如语言模型、多模态模型）。
小写字母开头，支持字母、数字、短横线（-），语义化描述算子功能
⚠️
功能标识
功能标识用于描述该镜像所提供的核心 AI 能力，将作为标签展示在镜像市场卡片中，帮助使用者快速理解镜像用途。命名规则：小写字母、数字及连字符 -，长度限制 30 字符。建议使用"动词-对象"的命名方式，例如：object-detector（目标检测）、text-classifier（文本分类）、image-segmenter（图像分割）、speech-recognizer（语音识别）。
不需要 GPU 资源
GPU 资源说明
此选项作为镜像的环境标签，用于标识该镜像是否需要 GPU 资源。开启后镜像标记为「需要 GPU」，在申请资源时系统将根据此环境标签判断是否需要申请 GPU 算力资源（而非直接分配 GPU）。默认关闭，请根据镜像实际运行需求设置。

⚠️
版本号命名规范
版本号遵循语义化版本规范（Semantic Versioning）。主版本号（Major）：当进行不兼容的 API 修改时递增；次版本号（Minor）：当新增向下兼容的功能名称时递增；修订版本号（Patch）：当进行向下兼容的问题修复时递增。每段支持字母、数字组合（如 1、0、3-beta），生成的完整版本格式为 Major.Minor.Patch，例如 1.0.0。
地址： registry.example.com/<category>/<name>:1.0.0
镜像地址规则
镜像地址由系统根据已填写的镜像分类、镜像名称和版本号自动拼接生成，格式为 registry.example.com/<分类>/<名称>:<版本>。此地址为只读预览，不可手动编辑。镜像创建成功后，其他用户可通过该地址从镜像仓库中拉取使用。

镜像描述
描述信息用于补充说明镜像的详细情况，帮助其他用户了解该镜像的用途与使用方式。建议填写以下内容：基础环境（如操作系统、CUDA 版本）、核心依赖库及版本号、适用业务场景、特殊配置说明（如环境变量、挂载路径）。选填，限制 500 字符。

文件上传说明
上传的压缩包需为完整的 Docker 镜像导出文件。在本地通过 docker save <image> | gzip > image.tar.gz 命令生成压缩包后上传。支持 .tar.gz 和 .tgz 格式，单个文件不超过 10GB。上传过程中会展示实时进度，上传完成后镜像将自动进入安全扫描流程，构建状态可在「镜像管理」页面查看。

上传说明
1
准备镜像文件
在本地通过 docker save <image> | gzip > image.tar.gz 导出镜像压缩包。
2
填写基础信息
在左侧表单中填写镜像名称、分类、功能标识、版本号和描述信息。
3
上传压缩包
点击左侧上传区域或将 .tar.gz 文件拖拽到上传区域，等待上传完成。
4
保存镜像
确认信息无误后点击"保存镜像"按钮，镜像将保存为「未构建」状态。之后可在镜像管理页面进入详情页触发构建。
💡 保存后镜像不会自动构建。请前往「镜像管理」→ 点击镜像卡片「详情」→ 点击「构建镜像」按钮触发构建。

确认信息无误后将保存镜像信息。保存后镜像状态为「未构建」，您可在镜像管理页面进入详情页触发构建。
